Get started / Overview

Welcome to KSeF Hub

One ledger for every invoice your company touches. KSeF Hub sits above Poland's KSeF protocol and absorbs everything else: foreign expenses, email drops, manual entry, API ingest. Built for companies that want a single source of truth across sources, users, and systems.

New here? Start with the Quickstart -- you'll connect KSeF, pull your first 30 days of invoices, and hit the API in under 10 minutes.

Quickstart

You'll need: a valid Polish NIP, a KSeF qualified certificate, and an admin email.

1. Create a workspace

Sign up at ksef-hub.pl/register. One workspace per company. Invite your team and accountant later.

2. Connect KSeF

Upload your qualified certificate in Settings -> Certificates. We handle XAdES signing, session tokens, and rotation. No code required.

# Certificate upload via API (alternative to UI) curl -X POST https://api.ksef-hub.pl/v1/certificates \ -H "Authorization: Bearer ksef_live_..." \ -F "file=@qualified-cert.p12" \ -F "passphrase=..."

3. Trigger the first sync

Hub pulls the last 30 days by default. After that, sync runs every 15 minutes in the background. You can force a sync from Dashboard -> Sync now.

4. Query the ledger

curl https://api.ksef-hub.pl/v1/invoices?limit=10 \ -H "Authorization: Bearer ksef_live_..."

Concepts

The ledger

The ledger is the single table of invoices + expenses for a workspace. Every row has a source field (ksef, email, upload, api, manual) but otherwise shares the same shape. Filters, permissions, and exports operate on the ledger as a whole.

Sources

  • ksef -- pulled via XAdES-signed API; full FA(3) XML preserved.
  • email -- forwarded to drop@yourco.ksef-hub.pl; OCR'd on ingest.
  • upload -- drag a PDF into the admin; parsed and filed.
  • api -- posted directly via POST /invoices.
  • manual -- entered by hand in the admin.

Permissions

Three roles: owner, member, accountant. Accountants see everything read-only by default; owners can grant edit + approve. Per-invoice visibility can be scoped with labels.

Guide: KSeF sync

Sync runs every 15 minutes. Each run signs a request with your certificate, fetches the delta, parses FA(3) XML, preserves signatures, and upserts into the ledger. Failed syncs retry with exponential backoff; after 6 failures we send a notification.

Certificate expiry. We email at 90, 30, and 7 days before expiry. Upload a replacement any time -- the old cert stays valid until you mark it superseded.

API reference

Base URL: https://api.ksef-hub.pl/v1. All requests require a Bearer token. Rate limit: 120 req/min per key.

Invoices

GET/invoicesList invoices, newest first.

Webhooks

Subscribe to ledger events. Payloads are signed with HMAC-SHA256; verify the X-KSeF-Signature header.

# Example: invoice.created { "event": "invoice.created", "created_at": "2026-04-18T09:02:11Z", "data": { "id": "inv_8u3f1", "source": "ksef", "number": "FV/2026/04/0142", "brutto": "1224.00", "currency": "PLN" } }
